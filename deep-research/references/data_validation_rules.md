# 数据验证规则参考

本文档定义了数据验证的标准规则、来源可靠性评估标准和数据冲突处理流程。

## 数据验证标准

### 1. 数据准确性验证

#### 1.1 交叉验证规则

```markdown
## 多源验证要求

### 关键数据
- 必须至少 3 个独立来源验证
- 优先使用官方数据源
- 记录所有验证来源

### 一般数据
- 至少 2 个来源验证
- 可使用专业平台数据
- 标注来源数量

### 估算数据
- 标明为估算值
- 说明估算方法
- 提供置信区间
```

#### 1.2 数据质量检查清单

```markdown
## 检查项目

### 完整性
- [ ] 所有关键指标是否都有数据
- [ ] 数据是否有时间戳
- [ ] 是否有缺失值说明

### 一致性
- [ ] 同一指标在不同来源是否一致
- [ ] 同一时期的数据是否可比
- [ ] 单位是否统一

### 准确性
- [ ] 数据是否符合逻辑
- [ ] 计算是否正确
- [ ] 是否有明显的异常值

### 时效性
- [ ] 数据是否在有效期内
- [ ] 是否使用最新数据
- [ ] 是否标注数据时间
```

### 2. 来源可靠性分级

#### 2.1 分级标准

```markdown
## 来源可靠性等级

| 等级 | 名称 | 说明 | 示例 |
|------|------|------|------|
| A | 官方机构 | 政府网站、国际组织、官方统计 | 国家统计局、WHO、央行 |
| B | 权威媒体 | 主流财经媒体、知名媒体 | 财经网、路透社、华尔街日报 |
| C | 专业平台 | 行业门户、专业数据平台 | 36氪、东方财富、行业协会 |
| D | 一般来源 | 博客、论坛、自媒体 | 个人博客、知乎、雪球 |
```

#### 2.2 等级评估标准

```markdown
## A级评估标准

### 必须满足
- [ ] .gov.cn / .org.cn 域名
- [ ] 官方机构发布
- [ ] 有明确发布时间
- [ ] 可追溯原始数据

### 加分项
- [ ] 数据经过审计
- [ ] 有详细方法论说明
- [ ] 国际组织认可

## B级评估标准

### 必须满足
- [ ] 知名媒体或机构
- [ ] 有专业编辑审核
- [ ] 引用来源可查

### 减分项
- [ ] 存在广告内容
- [ ] 可能存在利益冲突

## C级评估标准

### 必须满足
- [ ] 行业相关平台
- [ ] 有明确数据来源
- [ ] 更新频率合理

## D级评估标准

### 需要注意
- [ ] 需谨慎使用
- [ ] 必须与其他来源交叉验证
- [ ] 标注来源可靠性等级
```

### 3. 数据冲突处理

#### 3.1 冲突检测

```markdown
## 冲突类型

### 数值冲突
- 同一指标不同来源数值差异 > 10%
- 单位不统一导致的误解
- 时间范围不一致

### 时间冲突
- 数据时效性不同
- 统计周期不同
- 事件时间线冲突

### 定义冲突
- 指标定义不同
- 统计口径不同
- 样本范围不同
```

#### 3.2 冲突解决流程

```markdown
## 处理流程

### Step 1: 识别冲突
- 记录所有冲突数据点
- 标注来源和数值
- 识别冲突类型

### Step 2: 分析原因
- 检查数据定义是否一致
- 检查时间范围是否重叠
- 检查统计口径是否相同

### Step 3: 优先级判断
1. 官方来源 > 非官方来源
2. 一手来源 > 二手来源
3. 最新数据 > 历史数据
4. 权威媒体 > 一般媒体

### Step 4: 选择数据
- 采用高优先级来源
- 说明选择原因
- 记录冲突解决过程

### Step 5: 标注说明
- 在报告中标注数据差异
- 说明采用的数据
- 提供原始数据对比表
```

#### 3.3 冲突处理示例

```markdown
## 示例：GDP增速数据冲突

### 冲突描述
- 来源A（国家统计局）：5.2%
- 来源B（某研究机构）：5.0%
- 来源C（国际货币基金组织）：4.8%

### 解决过程
1. 所有来源均为A级/权威机构
2. 检查发现：IMF预测 vs 统计局实际数据
3. 采用官方实际数据（5.2%）
4. 标注：IMF为预测值4.8%

### 报告表述
根据国家统计局数据，2024年中国GDP增长率为5.2%[1]，高于国际货币基金组织(IMF)此前预测的4.8%[2]。

### 参考文献
1. 国家统计局 - 2024年国民经济和社会发展统计公报 [A]
2. IMF - 世界经济展望报告 [A]
```

### 4. 数据质量评分

#### 4.1 评分维度

```markdown
## 评分维度及权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 数据完整性 | 25% | 关键数据的覆盖程度 |
| 来源可靠性 | 30% | 高等级来源占比 |
| 时效性 | 20% | 数据更新时间 |
| 一致性 | 15% | 多源数据吻合度 |
| 可验证性 | 10% | 可追溯性 |
```

#### 4.2 评分标准

```markdown
## 评分等级

### A级 (90-100分)
- 官方数据源占比 > 80%
- 关键数据完整
- 数据时效性强
- 可完全验证

### B级 (70-89分)
- 官方/权威源占比 > 60%
- 主要数据完整
- 数据时效性较好
- 基本可验证

### C级 (50-69分)
- 混合来源
- 部分数据缺失
- 时效性一般
- 验证受限

### D级 (<50分)
- 依赖一般来源
- 数据不完整
- 时效性差
- 难以验证
```
