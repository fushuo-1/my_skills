# 硬件调试指南

## 一、调试方法论

### 1. 调试基本原则

**分模块调试**：
```
调试顺序：
1. 电源模块 → 验证供电正确
2. 时钟模块 → 验证时钟信号
3. 复位模块 → 验证复位时序
4. 核心模块 → 验证主芯片工作
5. 外设模块 → 逐个验证接口功能
```

**信号流向法**：
- 从输入到输出逐级检查
- 定位故障点后向前追溯
- 确认正常后向后验证

### 2. 常见故障分类

| 故障类型 | 典型表现 | 排查优先级 |
|----------|---------|-----------|
| 开路 | 无信号/电压异常 | 高 |
| 短路 | 电源异常/过热 | 高 |
| 参数错误 | 功能异常 | 中 |
| 时序错误 | 间歇性故障 | 中 |
| 干扰 | 随机故障 | 低 |

## 二、测试仪器使用

### 1. 示波器

**带宽选择**：
```
正弦波：带宽 ≥ 3 × 信号频率
方波：带宽 ≥ 5 × 信号频率

例：100MHz时钟信号
推荐带宽：≥ 500MHz
```

**探头选择**：
| 探头类型 | 带宽 | 负载效应 | 适用场景 |
|----------|------|---------|----------|
| 无源探头 | ≤500MHz | 高(10MΩ//10pF) | 通用测量 |
| 有源探头 | ≤4GHz | 低(1MΩ//<1pF) | 高速信号 |
| 差分探头 | ≤1GHz | 中 | 差分信号 |
| 电流探头 | ≤100MHz | - | 电流测量 |

**触发设置**：
```
边沿触发：常规信号测量
脉宽触发：捕获异常脉冲
超时触发：捕获丢失脉冲
序列触发：复杂时序分析
```

### 2. 逻辑分析仪

**采样率选择**：
```
采样率 ≥ 4 × 信号频率（推荐）

例：SPI时钟10MHz
采样率：≥ 40MSa/s
```

**协议解码**：
- I2C：识别地址、数据、ACK/NACK
- SPI：显示MOSI/MISO数据
- UART：显示发送/接收数据
- CAN：显示帧ID、数据

### 3. 网络分析仪

**S参数测量**：
```
S11：反射系数（回波损耗）
S21：传输系数（插入损耗）
S12：反向传输
S22：输出反射

典型测量：
- 阻抗测量：S11 → Z = Z0 × (1+S11)/(1-S11)
- 滤波器特性：S21
- 传输线特性：S11, S21
```

### 4. 频谱分析仪

**EMI诊断**：
```
设置：
- 频率范围：30MHz - 1GHz
- 分辨率带宽：120kHz (CISPR)
- 检波方式：准峰值/峰值

诊断步骤：
1. 扫描全频段
2. 定位超标频点
3. 近场探头定位源头
4. 分析谐波关系
```

## 三、电源调试

### 1. 电源基础检查

**电压检查**：
```
检查项目：
1. 输出电压精度
2. 输出纹波
3. 负载调整率
4. 线性调整率

测量方法：
- 万用表测量直流分量
- 示波器AC耦合测纹波
- 纹波测量带宽限制20MHz
```

**上电时序检查**：
```
时序要求：
- 核心电压先上
- I/O电压后上
- 时序间隔：1-10ms（典型）

验证方法：
- 示波器多通道同时测量
- 检查是否符合规格
```

### 2. 电源故障排查

**无输出电压**：
```
排查步骤：
1. 检查输入电压是否正常
2. 检查使能信号是否有效
3. 检查反馈网络
4. 检查输出是否短路
5. 检查电感/二极管
```

**输出电压偏低**：
```
可能原因：
1. 负载过重（电流超限）
2. 反馈电阻错误
3. 输入电压过低
4. 器件故障

排查方法：
1. 测量负载电流
2. 检查反馈电阻值
3. 检查输入电压
4. 更换电源IC
```

**输出纹波过大**：
```
可能原因：
1. 输出电容ESR过大
2. 电感值不合适
3. PCB布局不当
4. 负载电流突变

解决方案：
1. 更换低ESR电容
2. 调整电感值
3. 优化PCB布局
4. 增加输出电容
```

### 3. DC-DC调试要点

**稳定性检查**：
```
方法：负载阶跃响应测试

步骤：
1. 空载到满载阶跃
2. 观察输出电压响应
3. 检查是否有过冲/振铃

判据：
- 过冲：< 5% 输出电压
- 恢复时间：< 100µs
- 无持续振荡
```

**效率测量**：
```
效率 = Pout / Pin × 100%

测量方法：
- 输入：电压表+电流表
- 输出：电子负载

影响因素：
- 开关频率
- 电感DCR
- MOSFET导通电阻
- 驱动损耗
```

## 四、时钟与复位调试

### 1. 晶振调试

**起振检查**：
```
检查项：
1. 振荡频率是否正确
2. 振荡幅度是否足够
3. 波形是否正常（正弦/近似正弦）

典型参数：
- 频率精度：±30ppm（常规）
- 幅度：0.8-1.2Vpp（CMOS输出）
- 上升/下降沿：取决于频率
```

**晶振故障排查**：
```
不起振原因：
1. 负载电容不匹配
2. 晶振损坏
3. 芯片OSC引脚问题
4. PCB布局不当

排查步骤：
1. 检查负载电容值
2. 更换晶振
3. 测量OSC引脚
4. 检查布局
```

### 2. 时钟信号测量

**时钟质量评估**：
```
测量项：
1. 频率精度
2. 占空比
3. 上升/下降时间
4. 抖动

测量方法：
- 频率：频率计/示波器
- 占空比：示波器
- 抖动：示波器直方图
```

### 3. 复位调试

**复位信号检查**：
```
检查项：
1. 上电复位时序
2. 复位脉冲宽度
3. 复位释放时序
4. 手动复位功能

典型时序：
- 上电复位：≥ 1ms
- 手动复位：≥ 100µs
- 释放延迟：适当延时
```

## 五、通信接口调试

### 1. I2C调试

**波形检查**：
```
检查项：
1. SCL/SDA电平是否正确
2. 起始/停止条件是否正确
3. ACK/NACK是否正确
4. 时序是否符合规格

常见问题：
- 无ACK：地址错误/从机无响应
- 数据错误：时序问题
- 通信中断：上拉电阻问题
```

**时序验证**：
```
标准模式(100kHz)：
- SCL低电平：≥ 4.7µs
- SCL高电平：≥ 4.0µs
- 数据建立时间：≥ 250ns
- 数据保持时间：≥ 0ns

快速模式(400kHz)：
- SCL低电平：≥ 1.3µs
- SCL高电平：≥ 0.6µs
- 数据建立时间：≥ 100ns
- 数据保持时间：≥ 0ns
```

### 2. SPI调试

**模式配置**：
```
CPOL=0, CPHA=0: 模式0（最常用）
CPOL=0, CPHA=1: 模式1
CPOL=1, CPHA=0: 模式2
CPOL=1, CPHA=1: 模式3

确认主从模式一致！
```

**信号检查**：
```
检查项：
1. CS片选是否正确
2. SCK时钟是否正常
3. MOSI数据是否发送
4. MISO数据是否接收

常见问题：
- 数据错误：模式不匹配
- 无响应：CS未有效
- 时序问题：时钟频率过高
```

### 3. UART调试

**波特率验证**：
```
测量方法：
1. 示波器测量位宽
2. 计算波特率 = 1/位宽
3. 对比设定值

误差计算：
误差 = |实际-设定|/设定 × 100%
要求：误差 < 2%
```

**通信故障排查**：
```
常见问题：
1. 无数据：检查TX/RX连接
2. 乱码：波特率错误
3. 数据丢失：缓冲区溢出

排查步骤：
1. 示波器检查TX波形
2. 确认波特率一致
3. 检查数据位/停止位/校验位
4. 检查流控配置
```

## 六、故障诊断技巧

### 1. 信号完整性诊断

**反射问题**：
```
现象：信号振铃、过冲
原因：阻抗不匹配

诊断：
1. 示波器观察波形
2. 计算反射系数
3. 检查端接电阻

解决：
1. 添加串联端接
2. 调整走线阻抗
3. 更换驱动器
```

**串扰问题**：
```
现象：信号干扰、误码
原因：走线间距过小

诊断：
1. 近端/远端测量
2. 检查走线间距
3. 仿真验证

解决：
1. 增加走线间距
2. 增加地线隔离
3. 减小平行长度
```

### 2. 电源完整性诊断

**电源噪声**：
```
现象：电源纹波大、功能异常
原因：去耦不足、PDN阻抗高

诊断：
1. 示波器测量纹波
2. 网络分析仪测PDN阻抗
3. 仿真对比

解决：
1. 增加去耦电容
2. 优化电容布局
3. 改善电源平面
```

### 3. EMC问题诊断

**辐射发射超标**：
```
诊断流程：
1. 频谱分析仪定位频点
2. 近场探头定位源头
3. 分析耦合路径
4. 制定整改方案

常见源头：
- 晶振谐波
- 高速信号
- 电源开关
- 接口信号
```
